关于银行流水文件有三种格式，且处理逻辑不一样，要区分开。具体实践是：先选择模式，有三个按钮供用户选择 ，分别是：”银行“”支付宝“”微信“，不同选项对应执行逻辑不同。后面具体说明。结果文件要求格式为.xlsx， 结果文件命名为运行结束时间戳，结果文件内容是完整保留银行流水原始数据，新增预收款关联信息列，匹配失败行对应列留空， 结果文件存储路径为银行流水文件所在文件夹。


关于”银行“模式：
表头所在行号：16，数据起始行号：17；
匹配规则更新为：
1. 只和预收款文件中字段”*收款账户.名称 # settleaccount.name“ 是 ”中信银行5031“的行做匹配
2. 接着区分，分两步：
2.1先判断银行流水对应字段“对方账户名称”是中文的行：
2.1.1 必选匹配字段1：
    - 银行流水对应字段：“贷方发生额”
    - 预收款对应字段：“*收款金额 # amount”
    - 匹配逻辑：数值相等（金额类）
2.1.2 必选匹配字段2：
    - 银行流水对应字段：交易日期
    - 预收款对应字段：“*单据日期 # billdate”
    - 匹配逻辑：数值相等（日期类）
2.1.3 必选匹配字段3：
    - 银行流水对应字段：”对方账户名称“
    - 预收款对应字段：“*结算客户.名称 # itemclass.name”
    - 匹配逻辑：名字相同   
2.1.4 必选匹配字段4：
    -银行流水对应字段：“币种”
    - 预收款对应字段：“*币别.名称 # currency.name” 
    - 匹配逻辑：名称一致都是人民币
2.1.5 特殊匹配说明：
    - 金额匹配是否允许误差：否
    - 账号匹配是否区分大小写/特殊字符：否
2.2 再判断银行流水对应字段“对方账户名称”是英文的行：
2.2.1必选匹配字段1：
    - 银行流水对应字段：“贷方发生额”
    - 预收款对应字段：“*收款金额 # amount”“*汇率 # exchangerate”
    - 匹配逻辑：贷方发生额=round（收款金额 * 汇率，2）
2.2.2必选匹配字段2：
    - 预收款对应字段：“*币别.名称 # currency.name” 
    - 匹配逻辑：*币别.名称 # currency.name是美元
3. 信息填充规则为：
匹配成功后，需将预收款中哪些字段信息，拷贝至银行流水文件的哪一列，需明确对应关系：
1. 预收款文件字段名称：“*单据编号 # billno”，所对应的数据，拷贝到银行流水的第O列。注：银行流水文件中O列表头为“单据编号”。
2. 预收款文件字段名称：“*结算客户.名称 # itemclass.name”，所对应的数据，拷贝到银行流水的第P列。注：银行流水文件中P列表头为“客户”。
3. 预收款文件字段名称：“*订单编号 # srcbillno”，所对应的数据，拷贝到银行流水的第Q列。注：银行流水文件中Q列表头为“订单号”。


关于”支付宝“工作表：
数据所在工作表名称：支付宝，表头所在行号：3， 数据起始行号：4
